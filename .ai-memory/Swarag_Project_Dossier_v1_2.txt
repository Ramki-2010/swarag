=========================================================
1. WHAT IS SWARAG
=========================================================

Swarag identifies Carnatic ragas from audio using a deterministic
DSP pipeline (not deep learning). It extracts pitch features and
matches them against known raga signatures.

The goal is to codify Carnatic musical grammar into computation.
Raga identity is structured grammar, not statistical coincidence.

Current: 7 ragas modeled (70 clips), 2 more staged.
Target: 72 Melakartas, janya ragas, live inference, Android app.

=========================================================
2. PIPELINE
=========================================================

Audio -> Vocal Isolation (Demucs/stems) -> Pitch Extraction (pYIN)
-> Tonic Estimation -> Normalization (cents) -> 72-bin PCD +
Directional Dyads -> IDF x Variance Scoring -> Guardrails -> Output

Key parameters:
  - 72-bin PCD (17 cents per bin)
  - Directional dyads with ALPHA=0.01 Laplace smoothing
  - IDF x Variance weighted dot-product
  - PCD_WEIGHT=0.8, DYAD_WEIGHT=0.2 (global default)
  - PER_RAGA_WEIGHTS: Bhairavi=(0.5, 0.5)
  - MIN_CLIPS_PER_RAGA=5 guardrail
  - MAX_DURATION_SEC=360 (6-minute cap)

Details: .ai-memory/architecture.md

7. KEY DECISIONS
=========================================================

- Deterministic DSP (small dataset, explainability, control)
- 72-bin PCD (17 cents/bin resolves microtonal swaras)
- IDF x Variance weighting (downweights common swaras)
- PCD-heavy weights 0.8/0.2 (PCD more stable than dyads at current data)
- Directional dyads (Carnatic ragas are asymmetric)
- MIN_CLIPS_PER_RAGA=5 guardrail (thin-data ragas poison models)
- Sandbox-first workflow (never edit production untested)
- UNKNOWN over wrong certainty (margin guardrails)
- Per-raga weight override for Bhairavi (0.5/0.5) -- validated, locked
- Honest baselines over inflated numbers (L-041)
- Per-raga weight overrides are a trade for most ragas (L-042)
  Exception: Bhairavi override works because its dyads ARE distinctive

=========================================================
8. RISKS
=========================================================

- Abhogi/Kalyani parent-janya overlap makes PCD-only separation impossible
- Mohanam model weak (20% LOO) -- needs diverse clips
- Kamboji only 3 real clips after Harikambhoji cleanup
- Harikambhoji/Kamboji parent-janya confusion can recur with other ragas
- No absolute OOD score floor (margin-only detection)
- Silent failures: except-Exception hides crashes as UNKNOWN
- Constants must stay synchronized across scripts

Details: .ai-memory/bugs.md, .ai-memory/lessons.md
